# GRAMGPT Production Deployment Guide

## 📋 Prerequisites

- Docker & Docker Compose installed
- Domain name with DNS pointing to your server IP
- SSL certificate (auto-generated by Traefik via Let's Encrypt)
- All environment variables filled in `.env.prod`

## 🚀 Quick Start

### 1. Clone & Configure

```bash
cd /opt/gramgpt
cp .env.prod.example .env.prod
nano .env.prod  # Fill in ALL values
```

### 2. Generate Secure Keys

```bash
# JWT Secret
openssl rand -hex 32

# Fernet Encryption Key
python3 -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"

# Redis Password
openssl rand -base64 32
```

### 3. Deploy

```bash
docker-compose -f docker-compose.prod.yml up --build -d
```

### 4. Verify

```bash
# Check all services are running
docker-compose -f docker-compose.prod.yml ps

# View logs
docker-compose -f docker-compose.prod.yml logs -f api

# Test API
curl https://yourdomain.com/docs
```

## 🔄 CI/CD Pipeline

### GitHub Actions Setup

1. Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy to Production

on:
  push:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.10'
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Run tests
        run: pytest --cov=src
  
  deploy:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Deploy to server
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          key: ${{ secrets.SSH_PRIVATE_KEY }}
          script: |
            cd /opt/gramgpt
            git pull origin main
            docker-compose -f docker-compose.prod.yml up --build -d
            docker system prune -f
```

2. Add secrets to GitHub repo:
   - `SERVER_HOST`: Your server IP
   - `SERVER_USER`: SSH username
   - `SSH_PRIVATE_KEY`: Private SSH key

## 💾 Backup Strategy

### Daily Database Backup

```bash
#!/bin/bash
# backup.sh
DATE=$(date +%Y%m%d_%H%M%S)
docker exec gramgpt-db pg_dump -U gramgpt_user gramgpt_prod > /backups/db_$DATE.sql
find /backups -name "db_*.sql" -mtime +7 -delete
```

### Cron Job

```bash
crontab -e
# Add: 0 2 * * * /opt/gramgpt/backup.sh
```

### Restore from Backup

```bash
cat db_20250115_020000.sql | docker exec -i gramgpt-db psql -U gramgpt_user -d gramgpt_prod
```

## 🛠️ Troubleshooting

### Common Issues

#### 1. SSL Certificate Not Generating

```bash
# Check Traefik logs
docker-compose -f docker-compose.prod.yml logs traefik

# Verify DNS is pointing correctly
dig yourdomain.com

# Force renewal
docker-compose -f docker-compose.prod.yml stop traefik
rm -rf letsencrypt/acme.json
docker-compose -f docker-compose.prod.yml up -d traefik
```

#### 2. Database Connection Failed

```bash
# Check if postgres is healthy
docker-compose -f docker-compose.prod.yml ps postgres

# View postgres logs
docker-compose -f docker-compose.prod.yml logs postgres

# Test connection
docker exec -it gramgpt-db psql -U gramgpt_user -d gramgpt_prod
```

#### 3. Celery Worker Not Processing Tasks

```bash
# Check worker logs
docker-compose -f docker-compose.prod.yml logs celery-worker

# Inspect queue
docker exec -it gramgpt-redis redis-cli -a $REDIS_PASSWORD keys '*'

# Restart worker
docker-compose -f docker-compose.prod.yml restart celery-worker
```

#### 4. High Memory Usage

```bash
# Check container stats
docker stats

# Limit worker concurrency in docker-compose.yml
# Change: --concurrency=4 to --concurrency=2
```

### Performance Tuning

#### API Workers

Adjust based on CPU cores:
```yaml
command: uvicorn src.api.main:app --host 0.0.0.0 --port 8000 --workers $(nproc)
```

#### Celery Concurrency

For I/O bound tasks:
```bash
celery -A src.core.celery_app worker --concurrency=8
```

For CPU bound tasks:
```bash
celery -A src.core.celery_app worker --concurrency=2
```

#### Database Connection Pool

In `src/db/database.py`:
```python
engine = create_engine(
    DATABASE_URL,
    pool_size=20,
    max_overflow=40,
    pool_recycle=3600
)
```

## 🆘 Emergency Procedures

### Rollback to Previous Version

```bash
git checkout <previous-commit>
docker-compose -f docker-compose.prod.yml up --build -d
```

### Database Emergency Restore

```bash
# Stop application
docker-compose -f docker-compose.prod.yml stop api celery-worker

# Restore database
cat latest_backup.sql | docker exec -i gramgpt-db psql -U gramgpt_user -d gramgpt_prod

# Restart application
docker-compose -f docker-compose.prod.yml start api celery-worker
```

### DDoS Mitigation

1. Enable Cloudflare proxy (orange cloud)
2. Configure rate limiting in Traefik
3. Temporarily block suspicious IPs
